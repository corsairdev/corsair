## Setup

<Steps>

<Step>

### Install

```bash
pnpm install corsair
```

</Step>

<Step>

### Generate your encryption key

Corsair encrypts every stored credential with a Key Encryption Key (KEK). Click to generate one and copy it into your `.env`:

<KekGenerator />

<Callout type="warn" title="Keep this key safe">
If you lose it, you lose access to all stored credentials. Treat it like a root password.
</Callout>

</Step>

<Step>

### Migrate the database

Corsair needs five tables. Run this once, then you're done.

<Tabs items={['SQLite', 'PostgreSQL']}>
  <Tab value="SQLite">

Install the driver, then run the migration:

```bash
pnpm install better-sqlite3
pnpm install --save-dev @types/better-sqlite3
```

</Tab>
  <Tab value="PostgreSQL">

Install the driver, then run the migration:

```bash
pnpm install pg
pnpm install --save-dev @types/pg
```

</Tab>
</Tabs>

    entity_type TEXT NOT NULL,
    version TEXT NOT NULL,
    data TEXT NOT NULL DEFAULT '{}',

CREATE TABLE IF NOT EXISTS corsair_events (
    id TEXT PRIMARY KEY,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL,

    tenant_id TEXT NOT NULL DEFAULT 'default',

<Step>

### Create `corsair.ts`

This is the one file that wires everything together. Create it at your project root:

<Tabs items={['SQLite', 'PostgreSQL']}>
<Tab value="SQLite">

```ts title="corsair.ts"
import Database from 'better-sqlite3';
import { createCorsair, github } from 'corsair';

const db = new Database('corsair.db');

export const corsair = createCorsair({
    plugins: [github()],
    database: db,
    kek: process.env.CORSAIR_KEK!,
});
```

</Tab>
<Tab value="PostgreSQL">

```ts title="corsair.ts"
import { Pool } from 'pg';
import { createCorsair, github } from 'corsair';

const db = new Pool({ connectionString: process.env.DATABASE_URL });

export const corsair = createCorsair({
    plugins: [github()],
    database: db,
    kek: process.env.CORSAIR_KEK!,
});
```

</Tab>
</Tabs>

You'll add more plugins here later — `slack()`, `linear()`, `gmail()` — same pattern, just append to the array.

<Accordions>
<Accordion title="CORSAIR_KEK not loading?">

If `process.env.CORSAIR_KEK` is `undefined` at runtime, your `.env` file isn't being picked up. Install dotenv and add one line at the top of `corsair.ts`:

```bash
pnpm install dotenv
```

```ts title="corsair.ts"
import 'dotenv/config'; // add this line
import Database from 'better-sqlite3';
// ...
```

</Accordion>
</Accordions>

</Step>

<Step>

### Add your GitHub token and run setup

1. Go to **[GitHub → Settings → Developer settings → Personal access tokens (classic)](https://github.com/settings/tokens)**
2. Click **Generate new token (classic)**
3. Select scopes: `repo`, `read:org`, `read:user`
4. Copy the token — it starts with `ghp_`

Run setup once with the Corsair CLI:

```bash
npx corsair setup --github api_key=$GITHUB_TOKEN --backfill
```

Corsair stores your token, creates the database rows, and fetches your GitHub data locally. Your agent can now query it instantly — no API call needed every time.

</Step>

</Steps>

## Interact with Corsair

Two ways to use it — pick one.

<Tabs items={['MCP / Agent', 'Direct (Hardcode)']}>

<Tab value="MCP / Agent">

Let the agent discover and call endpoints on its own. You prompt in plain English — Corsair handles the rest.

Install the MCP adapter alongside your preferred framework:

<Tabs items={['Anthropic SDK', 'Claude Agent SDK', 'OpenAI Agents']}>

<Tab value="Anthropic SDK">
```bash
pnpm install @corsair-dev/mcp @anthropic-ai/sdk
```

```ts title="agent.ts"
import Anthropic from '@anthropic-ai/sdk';
import { AnthropicProvider } from '@corsair-dev/mcp';
import { corsair } from './corsair';

async function main() {
    const provider = new AnthropicProvider();
    const tools = provider.build({ corsair });
    const client = new Anthropic();

    const message = await client.beta.messages.toolRunner({
        model: 'claude-sonnet-4-6',
        max_tokens: 4096,
        tools,
        messages: [{
            role: 'user',
            content: 'List my GitHub repos with the most open issues.',
        }],
    });

    for (const block of message.content) {
        if (block.type === 'text') console.log(block.text);
    }
}

main().catch(console.error);
```

</Tab>

<Tab value="Claude Agent SDK">

```bash
pnpm install @corsair-dev/mcp @anthropic-ai/claude-agent-sdk
```

```ts title="agent.ts"
import { createSdkMcpServer, query } from '@anthropic-ai/claude-agent-sdk';
import { ClaudeProvider } from '@corsair-dev/mcp';
import { corsair } from './corsair';

async function main() {
    const provider = new ClaudeProvider();
    const tools = await provider.build({ corsair });
    const server = createSdkMcpServer({ name: 'corsair', tools });

    const stream = query({
        prompt: 'List my GitHub repos with the most open issues.',
        options: {
            model: 'claude-opus-4-6',
            mcpServers: { corsair: server },
        },
    });

    for await (const event of stream) {
        if ('result' in event) process.stdout.write(event.result);
    }
}

main().catch(console.error);
```

</Tab>

<Tab value="OpenAI Agents">

```bash
pnpm install @corsair-dev/mcp @openai/agents
```

```ts title="agent.ts"
import { OpenAIAgentsProvider } from '@corsair-dev/mcp';
import { Agent, run, tool } from '@openai/agents';
import { corsair } from './corsair';

async function main() {
    const provider = new OpenAIAgentsProvider();
    const tools = provider.build({ corsair, tool });

    const agent = new Agent({
        name: 'corsair-agent',
        model: 'gpt-4.1',
        instructions:
            'You have access to Corsair tools. Use list_operations to discover ' +
            'available APIs, get_schema to understand arguments, and run_script ' +
            'to execute them.',
        tools,
    });

    const result = await run(agent, 'List my GitHub repos with the most open issues.');
    console.log(result.finalOutput);
}

main().catch(console.error);
```

</Tab>

</Tabs>

Your agent gets four tools automatically — regardless of which framework you use:

| Tool | What it does |
|---|---|
| `corsair_setup` | Check auth and get credential instructions |
| `list_operations` | Discover every available API endpoint |
| `get_schema` | Inspect parameters for a specific endpoint |
| `run_script` | Execute a JS snippet with `corsair` in scope |

**Copy this prompt and run it:**

```
Setup corsair. Then fetch my GitHub repositories, find the ones with the
most open issues, and give me a summary of the top 3 — what they're about,
how many issues each has, and what the most common issue types look like.
```

The agent calls `corsair_setup` to verify auth, `list_operations` to discover GitHub endpoints, then `run_script` to fetch and summarize. No code changes needed on your end.

See [MCP Adapters](/mcp-adapters/claude-sdk) for Vercel AI SDK, Mastra, and more.

</Tab>

<Tab value="Direct (Hardcode)">

Call any endpoint from plain TypeScript — no agent, no framework required.

The pattern is always: **`corsair.[plugin].[api | db].[group].[method]()`**

**API — live calls to GitHub:**

```ts
// Get a specific repo
const repo = await corsair.github.api.repositories.get({
    owner: 'octocat',
    repo: 'hello-world',
});

console.log(repo.name); // "hello-world"
```

**DB — query what's already cached locally:**

After backfill runs (or after any API call), data lives in your database. Query it without touching the GitHub API:

```ts
// Every repo you've fetched — instant, no network
const repos = await corsair.github.db.repositories.search({});

// Filter in JS using camelCase data fields
const withIssues = repos.filter((r) => Number(r.data?.openIssuesCount ?? 0) > 0);

for (const repo of withIssues) {
    console.log(`${repo.data?.name} — ${repo.data?.openIssuesCount} open issues`);
}
```

</Tab>
