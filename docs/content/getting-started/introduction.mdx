Corsair is a unified SDK for integrating third-party services into TypeScript applications. Corsair provides credential security, tenant data isolation, and rate limiting out of the box. You can work with hundreds of integrations like Slack or Github. Use the same syntax, same types, and the same four database tables. Focus on building your product instead of writing the same boilerplate integration code.

```ts title="corsair.ts"
import { createCorsair } from "corsair";
import { Pool } from "pg";
import { slack, linear, github, gmail, resend } from "corsair/plugins";

export const corsair = createCorsair({
    multiTenancy: true,
    database: new Pool({ connectionString: process.env.DATABASE_URL }),
    plugins: [
        slack(),
        linear(),
        github(),
        gmail(),
        resend(),
    ],
    kek: process.env.CORSAIR_KEK!,
});
```

## Quick Example

Once set up, calling APIs is straightforward:

```ts title="example.ts"
const tenant = corsair.withTenant("tenant_123");

// Send a Slack message
await tenant.slack.api.messages.post({
    channel: "C01234567",
    text: "Hello from Corsair!",
});

// Create a Linear issue
await tenant.linear.api.issues.create({
    title: "New feature request",
    teamId: "TEAM_ABC",
});
```

## Why Corsair?

### One SDK, Many Integrations

Learn one syntax, use it everywhere. No need to master each service's unique SDK quirks.

### Multi-Tenant by Default

Every operation is scoped to a tenant. Data isolation is automatic — no cross-contamination possible.

### Always Fresh Data

API responses are stored and automatically updated through webhooks. Your data is never stale.

### Four Tables Forever

Add 100 integrations — still just four database tables. No schema bloat.

### Fully Typed

Every API call, every response, every database query is fully typed. Your editor knows everything.
