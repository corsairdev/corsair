The Sentry plugin provides access to Sentry's API for managing events, issues, organizations, projects, releases, and teams.

<Note>
**Full Implementation:** For the complete, up-to-date list, see the [Sentry plugin source code](https://github.com/corsairdev/corsair/tree/main/packages/corsair/plugins/sentry).
</Note>

## Events

### get
`events.get`

Get an event by ID.

```ts
const event = await corsair.sentry.api.events.get({
    organizationSlug: "my-org", // [!code highlight]
    projectSlug: "my-project",
    eventId: "event-id",
});
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `organizationSlug` | `string` | Yes | Organization slug |
| `projectSlug` | `string` | Yes | Project slug |
| `eventId` | `string` | Yes | Event ID |

---

### list
`events.list`

List events for a project.

```ts
const events = await corsair.sentry.api.events.list({
    organizationSlug: "my-org", // [!code highlight]
    projectSlug: "my-project",
    limit: 25,
});
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `organizationSlug` | `string` | Yes | Organization slug |
| `projectSlug` | `string` | Yes | Project slug |
| `limit` | `number` | No | Max events to return |
| `cursor` | `string` | No | Pagination cursor |

## Issues

### get
`issues.get`

Get an issue by ID.

```ts
const issue = await corsair.sentry.api.issues.get({
    organizationSlug: "my-org", // [!code highlight]
    issueId: "issue-id",
});
```

---

### list
`issues.list`

List issues for a project.

```ts
const issues = await corsair.sentry.api.issues.list({
    organizationSlug: "my-org", // [!code highlight]
    projectSlug: "my-project",
    query: "is:unresolved",
    limit: 25,
});
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `organizationSlug` | `string` | Yes | Organization slug |
| `projectSlug` | `string` | No | Filter by project |
| `query` | `string` | No | Sentry search query |
| `limit` | `number` | No | Max issues to return |

---

### update
`issues.update`

Update an issue (resolve, assign, ignore, etc.).

```ts
await corsair.sentry.api.issues.update({
    organizationSlug: "my-org", // [!code highlight]
    issueId: "issue-id",
    status: "resolved",
});
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `organizationSlug` | `string` | Yes | Organization slug |
| `issueId` | `string` | Yes | Issue ID |
| `status` | `string` | No | `resolved`, `unresolved`, `ignored` |
| `assignedTo` | `string` | No | Username or team slug to assign |

---

### delete
`issues.delete`

Permanently delete an issue.

```ts
await corsair.sentry.api.issues.delete({
    organizationSlug: "my-org", // [!code highlight]
    issueId: "issue-id",
});
```

## Releases

### get
`releases.get`

Get a release by version ID.

```ts
const release = await corsair.sentry.api.releases.get({
    organizationSlug: "my-org", // [!code highlight]
    version: "1.0.0",
});
```

---

### list
`releases.list`

List releases for an organization.

```ts
const releases = await corsair.sentry.api.releases.list({
    organizationSlug: "my-org", // [!code highlight]
    projectSlug: "my-project",
});
```

---

### create
`releases.create`

Create a new release.

```ts
const release = await corsair.sentry.api.releases.create({
    organizationSlug: "my-org", // [!code highlight]
    version: "1.0.1",
    projects: ["my-project"],
});
```

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `organizationSlug` | `string` | Yes | Organization slug |
| `version` | `string` | Yes | Release version string |
| `projects` | `string[]` | Yes | Project slugs to associate |
| `ref` | `string` | No | Git ref (commit SHA, branch) |
| `url` | `string` | No | URL pointing to release info |

---

### update
