The Sentry plugin includes webhook handlers for Sentry issue, error, alert, and comment events.

<Info>
**New to Corsair?** Learn about [webhooks](/concepts/webhooks) and [hooks](/concepts/hooks).
</Info>

<Note>
**Full Implementation:** See the [Sentry plugin source code](https://github.com/corsairdev/corsair/tree/main/packages/corsair/plugins/sentry).
</Note>

## Setup

```ts app/api/webhook/route.ts
import { processWebhook } from "corsair";
import { corsair } from "@/server/corsair";

export async function POST(request: Request) {
    const headers = Object.fromEntries(request.headers);
    const body = await request.json();

    const result = await processWebhook(corsair, headers, body);
    return result.response;
}
```

Sentry webhooks are identified by both the `sentry-hook-signature` and `sentry-hook-resource` headers.

## Available Webhooks

### issues.created

Fires when a new issue is created in Sentry.

**Example:**

```ts corsair.ts
sentry({
    webhookHooks: {
        "issues.created": async (event) => {
            console.log("New issue:", event.data?.issue?.title);
            console.log("Project:", event.data?.issue?.project?.slug);
        },
    },
})
```

**Key fields:** `data.issue.id`, `data.issue.title`, `data.issue.culprit`, `data.issue.status`, `data.issue.project`, `actor`
