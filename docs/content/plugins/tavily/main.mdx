## Quick Start

Install the plugin:

```bash
pnpm install @corsair-dev/tavily
```

Add the Tavily plugin to your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from "corsair";
import { tavily } from "@corsair-dev/tavily";

export const corsair = createCorsair({
    plugins: [
        tavily(),
    ],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from "corsair";
import { tavily } from "@corsair-dev/tavily";

export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [tavily()],
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
</Tabs>

Once configured, you can search the web:

<Tabs>
<Tab title="Solo">
```ts
// Search the web
const results = await corsair.tavily.api.search({
    query: "latest AI research papers 2024",
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts
const tenant = corsair.withTenant("user_id");

// Search the web
const results = await tenant.tavily.api.search({
    query: "latest AI research papers 2024",
});
```
</Tab>
</Tabs>

## Authentication

### Supported Auth Types

The Tavily plugin supports:
- **[`api_key`](/content/concepts/api-key)** (default) - Use a Tavily API key

### Default Auth Type

If no `authType` is specified, the plugin defaults to **[`api_key`](/content/concepts/api-key)**.

### Configuring API Key Authentication

<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --tavily api_key=your-api-key
```
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --tavily api_key=their-api-key --tenant=user_abc123
```
</Tab>
</Tabs>

See [API Key Authentication](/content/concepts/api-key) for full setup details.

## Options

| Option | Type | Description |
|--------|------|-------------|
| `authType` | `'api_key'` | Authentication method (defaults to `'api_key'`) |
| `key` | `string` | API key (optional, uses database if not provided) |
| `hooks` | `object` | Endpoint hooks for custom logic |
| `errorHandlers` | `object` | Custom error handlers |
| `permissions` | `object` | Permission configuration for AI agent access |

### Hooks

```ts
tavily({
    hooks: {
        search: {
            before: async (ctx, input) => {
                console.log("Searching for:", input.query);
                return { ctx, input };
            },
        },
    },
})
```

See [Hooks](/concepts/hooks) for complete documentation.

### Error Handling

The plugin includes built-in error handlers for common scenarios. For complete documentation, see the [Error Handlers reference](/plugins/tavily/error-handlers).

## Usage

### Accessing the API

<Tabs>
<Tab title="Solo">
```ts
// Basic search
```

## Examples

### Example 1: AI Research Assistant

```ts inngest/functions.ts
export const researchTopic = inngest.createFunction(
    { id: "research-topic" },
    { event: "app/research-requested" },
    async ({ event }) => {
        const tenant = corsair.withTenant(event.data.tenantId);

        const results = await tenant.tavily.api.search({
            query: event.data.topic, // [!code highlight]
            search_depth: "advanced",
            include_answer: true, // [!code highlight]
            max_results: 5,
        });

        return {
            answer: results.data.answer,
            sources: results.data.results.map(r => r.url),
        };
    }
);
```

### Example 2: Real-time News Aggregation

```ts inngest/functions.ts
export const fetchNewsDigest = inngest.createFunction(
    { id: "fetch-news-digest" },
    { cron: "0 9 * * *" },
    async () => {
        const topics = ["AI", "TypeScript", "web development"];

        const searches = await Promise.all(
            topics.map(topic =>
                corsair.tavily.api.search({
                    query: `${topic} news today`, // [!code highlight]
                    topic: "news",
                    time_range: "day",
                    max_results: 3,
                })
            )
        );

        return searches.map((s, i) => ({
            topic: topics[i],
            results: s.data.results,
        }));
    }
);
```
