## Built-in Error Handlers

### Rate Limit Errors

**Triggers:** HTTP 429 status

**Behavior:**
- Maximum retries: **5**
- Respects `Retry-After` header
- Twitter's rate limits are per 15-minute window per endpoint

**Example:**

```ts
// Corsair handles rate limits automatically
const tweets = await corsair.twitterapiio.api.tweets.search({
    query: "typescript", // [!code highlight]
});
```

---

### Authentication Errors

**Triggers:** HTTP 401 status, invalid API key

**Behavior:**
- Maximum retries: **0** (no retries)
- Fails immediately

**How to Fix:**
1. Verify your twitterapi.io API key is valid
2. Check the key has not expired or been revoked

---

### Forbidden Errors

**Triggers:** HTTP 403 status, insufficient permissions

**Behavior:**
- Maximum retries: **0** (no retries)

**How to Fix:**
1. Verify your twitterapi.io plan supports the endpoint you're calling
2. Some endpoints require elevated API access

---

### Not Found Errors

**Triggers:** HTTP 404 status, tweet or user does not exist

**Behavior:**
- Maximum retries: **0** (no retries)

**How to Fix:**
1. Verify the tweet ID or user ID is correct
2. Check that the tweet has not been deleted or the account has not been suspended

---

### Network Errors

**Triggers:** Connection errors, timeouts

**Behavior:**
- Maximum retries: **3**
- Uses exponential backoff

---

## Custom Error Handlers

```ts corsair.ts
twitterapiio({
    errorHandlers: {
        DUPLICATE_TWEET: {
            match: (error) => {
                return error.message.includes("duplicate content"); // [!code highlight]
            },
            handler: async (error, context) => {
                console.log("Duplicate tweet detected — skipping");
                return {
                    maxRetries: 0,
                };
            },
        },
    },
})
```
