## Quick Start

Install the plugin:

```bash
pnpm install @corsair-dev/googlecalendar
```

Add the Google Calendar plugin to your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from "corsair";
import { googlecalendar } from "@corsair-dev/googlecalendar";

export const corsair = createCorsair({
    plugins: [
        googlecalendar({
            authType: "oauth_2",
        }),
    ],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from "corsair";
import { googlecalendar } from "@corsair-dev/googlecalendar";

export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [
        googlecalendar({
            authType: "oauth_2",
        }),
    ],
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
</Tabs>

Once configured, you can access the Google Calendar API through your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts
const events = await corsair.googlecalendar.api.events.getMany({
    calendarId: "primary",
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts
const tenant = corsair.withTenant("user_id");

const events = await tenant.googlecalendar.api.events.getMany({
    calendarId: "primary",
});
```
</Tab>
</Tabs>

## Authentication

### Supported Auth Types

The Google Calendar plugin uses OAuth 2.0 authentication exclusively.

- **[`oauth_2`](/content/concepts/oauth)** (default) - OAuth 2.0 authentication

### OAuth 2.0 Setup

<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --googlecalendar client_id=your-client-id client_secret=your-client-secret
pnpm corsair auth --plugin=googlecalendar
```

See [Get Credentials](/plugins/googlecalendar/get-credentials) for step-by-step instructions on setting up OAuth 2.0.
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --googlecalendar client_id=your-client-id client_secret=your-client-secret
```
</Tab>
</Tabs>

See [OAuth 2.0 Authentication](/content/concepts/oauth) for full setup including callback handling.

## Options

The Google Calendar plugin accepts the following configuration options:

| Option | Type | Description |
|--------|------|-------------|
| `authType` | `'oauth_2'` | Authentication method (defaults to `'oauth_2'`) |
| `credentials` | `object` | Credentials object with clientId and clientSecret |
| `hooks` | `object` | Endpoint hooks for custom logic |
| `webhookHooks` | `object` | Webhook hooks for event handling |

For complete documentation on getting credentials, see the [Plugin Credentials Guide](/guides/plugin-credentials#google-calendar).
