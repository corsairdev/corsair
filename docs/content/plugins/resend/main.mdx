## Quick Start

Install the plugin:

```bash
pnpm install @corsair-dev/resend
```

Add the Resend plugin to your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from "corsair";
import { resend } from "@corsair-dev/resend";

export const corsair = createCorsair({
    plugins: [
        resend(),
    ],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from "corsair";
import { resend } from "@corsair-dev/resend";

export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [resend()],
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
</Tabs>

Once configured, you can access the Resend API through your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts
await corsair.resend.api.emails.send({
    from: "hello@example.com",
    to: "user@example.com",
    subject: "Hello",
    html: "<p>Hello world!</p>",
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts
const tenant = corsair.withTenant("user_id");

await tenant.resend.api.emails.send({
    from: "hello@example.com",
    to: "user@example.com",
    subject: "Hello",
    html: "<p>Hello world!</p>",
});
```
</Tab>
</Tabs>

## Authentication

### Supported Auth Types

The Resend plugin uses API key authentication.

- **[`api_key`](/content/concepts/api-key)** (default) - API key authentication

### API Key Setup

<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --resend api_key=your-api-key
```

For webhook signature verification:

```bash
pnpm corsair setup --resend webhook_signature=your-webhook-secret
```

See [Get Credentials](/plugins/resend/get-credentials) for step-by-step instructions on obtaining an API key.
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --resend api_key=their-api-key --tenant=user_abc123
```

For webhook signature verification:

```bash
pnpm corsair setup --resend webhook_signature=your-webhook-secret
```
</Tab>
</Tabs>

See [API Key Authentication](/content/concepts/api-key) for full setup details.

## Options

The Resend plugin accepts the following configuration options:

| Option | Type | Description |
|--------|------|-------------|
| `authType` | `'api_key'` | Authentication method (defaults to `'api_key'`) |
| `key` | `string` | API key (optional, uses database if not provided) |
| `webhookSecret` | `string` | Webhook signing secret (optional) |
| `hooks` | `object` | Endpoint hooks for custom logic |
| `webhookHooks` | `object` | Webhook hooks for event handling |
| `errorHandlers` | `object` | Custom error handlers |

For complete documentation on getting credentials, see the [Plugin Credentials Guide](/guides/plugin-credentials#resend).
