## Quick Start

Install the plugin:

```bash
pnpm install @corsair-dev/gmail
```

Add the Gmail plugin to your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from "corsair";
import { gmail } from "@corsair-dev/gmail";

export const corsair = createCorsair({
    plugins: [
        gmail({
            authType: "oauth_2",
        }),
    ],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from "corsair";
import { gmail } from "@corsair-dev/gmail";

export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [
        gmail({
            authType: "oauth_2",
        }),
    ],
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
</Tabs>

Once configured, you can access the Gmail API through your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts
const messages = await corsair.gmail.api.messages.list({
    maxResults: 10,
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts
const tenant = corsair.withTenant("user_id");

const messages = await tenant.gmail.api.messages.list({
    maxResults: 10,
});
```
</Tab>
</Tabs>

## Authentication

### Supported Auth Types

The Gmail plugin uses OAuth 2.0 authentication exclusively.

- **`oauth_2`** (default) - OAuth 2.0 authentication

### OAuth 2.0 Setup

<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --gmail client_id=your-client-id client_secret=your-client-secret
pnpm corsair auth --plugin=gmail
```

See [Get Credentials](/plugins/gmail/get-credentials) for step-by-step instructions on setting up OAuth 2.0.
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --gmail client_id=your-client-id client_secret=your-client-secret
```
</Tab>
</Tabs>

See [OAuth 2.0 Authentication](/concepts/oauth) for full setup including callback handling.

## Options

The Gmail plugin accepts the following configuration options:

| Option | Type | Description |
|--------|------|-------------|
| `authType` | `'oauth_2'` | Authentication method (defaults to `'oauth_2'`) |
| `credentials` | `object` | Credentials object with clientId and clientSecret |
| `hooks` | `object` | Endpoint hooks for custom logic |
| `webhookHooks` | `object` | Webhook hooks for event handling |

For complete documentation on getting credentials, see the [Plugin Credentials Guide](/guides/plugin-credentials#gmail).
