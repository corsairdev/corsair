## Query API

All entities share the same query shape:

```ts
// Return all matching records
const results = await corsair.{{PLUGIN_CLI_NAME}}.db.{{ENTITY_1_KEY}}.search({
    data: { {{ENTITY_1_SEARCH_FIELD}}: 'value' },  // filter by any schema field
    limit: 100,                                      // optional
    offset: 0,                                       // optional, for pagination
});

// Return all records
const all = await corsair.{{PLUGIN_CLI_NAME}}.db.{{ENTITY_1_KEY}}.search({});
```

## Multi-tenancy

All queries are automatically scoped when you use `withTenant`:

```ts
const tenant = corsair.withTenant('user-123');

const results = await tenant.{{PLUGIN_CLI_NAME}}.db.{{ENTITY_1_KEY}}.search({
    data: { {{ENTITY_1_SEARCH_FIELD}}: 'value' },
});
```

Each tenant's data is fully isolated. See [Multi-Tenancy](/concepts/multi-tenancy).

## When data is synced

1. **API calls** — every `api.*` call syncs its response automatically
2. **Webhooks** — incoming webhook events update the relevant entity
3. **Backfill** — running `corsair setup --backfill` does an initial sync of all entities

To force a refresh, call the corresponding API endpoint:

```ts
// Re-sync {{ENTITY_1_DISPLAY_NAME_LOWER}}
await corsair.{{PLUGIN_CLI_NAME}}.api.{{ENTITY_1_KEY}}.list({});

// Then query the updated cache
const fresh = await corsair.{{PLUGIN_CLI_NAME}}.db.{{ENTITY_1_KEY}}.search({});
```
