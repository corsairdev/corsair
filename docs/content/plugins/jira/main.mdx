## Quick Start

Install the plugin:

```bash
pnpm install @corsair-dev/jira
```

Add the Jira plugin to your Corsair instance:

<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from "corsair";
import { jira } from "@corsair-dev/jira";

export const corsair = createCorsair({
    plugins: [jira()],
    database: db,
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from "corsair";
import { jira } from "@corsair-dev/jira";

export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [jira()],
    database: db,
    kek: process.env.CORSAIR_KEK!,
});
```
</Tab>
</Tabs>

## Authentication

### Supported Auth Types

- **[`api_key`](/content/concepts/api-key)** (default) - API token with email

### API Key Setup

<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --jira api_key=your-api-token
```

For webhook signature verification:

```bash
pnpm corsair setup --jira webhook_signature=your-webhook-secret
```

See [Get Credentials](/plugins/jira/get-credentials) for step-by-step instructions.
</Tab>
<Tab title="Multi-Tenant">
Enable `multiTenancy` in your Corsair config, then store each user's key separately:

```ts corsair.ts
export const corsair = createCorsair({
    multiTenancy: true,
    plugins: [jira()],
    kek: process.env.CORSAIR_KEK!,
});
```

```bash
pnpm corsair setup --jira api_key=their-api-token --tenant=user_abc123
```

For webhook signature verification:

```bash
pnpm corsair setup --jira webhook_signature=your-webhook-secret
```
</Tab>
</Tabs>

See [API Key Authentication](/content/concepts/api-key) for full setup details.

## Options

| Option | Type | Description |
|--------|------|-------------|
| `authType` | `'api_key'` | Authentication method |
| `key` | `string` | API token (optional, uses database if not provided) |
| `webhookSecret` | `string` | Webhook signing secret (optional) |
| `hooks` | `object` | Endpoint hooks for custom logic |
| `webhookHooks` | `object` | Webhook hooks for event handling |
| `errorHandlers` | `object` | Custom error handlers |

## Usage

<Tabs>
<Tab title="Solo">
```ts
// Search issues
const issues = await corsair.jira.api.issues.search({
    jql: "project = MYPROJ AND status = 'In Progress'",
});

// Create an issue
await corsair.jira.api.issues.create({
    fields: {
        project: { key: "MYPROJ" },
        summary: "Fix login bug",
        issuetype: { name: "Bug" },
    },
});

// Add a comment
await corsair.jira.api.comments.add({
    issueIdOrKey: "MYPROJ-123",
    body: "Investigating the issue now.",
});

// List projects
const projects = await corsair.jira.api.projects.list({});

// Get sprint issues
const sprintIssues = await corsair.jira.api.sprints.getIssues({
    sprintId: "sprint-id",
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts
const tenant = corsair.withTenant("workspace-123");

// Search issues
const issues = await tenant.jira.api.issues.search({
    jql: "project = MYPROJ AND status = 'In Progress'",
});

// Create an issue
await tenant.jira.api.issues.create({
    fields: {
        project: { key: "MYPROJ" },
        summary: "Fix login bug",
        issuetype: { name: "Bug" },
    },
});

// Add a comment
await tenant.jira.api.comments.add({
    issueIdOrKey: "MYPROJ-123",
    body: "Investigating the issue now.",
});

// List projects
const projects = await tenant.jira.api.projects.list({});
```
</Tab>
</Tabs>
