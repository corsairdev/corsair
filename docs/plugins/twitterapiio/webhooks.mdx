The TwitterAPI.io plugin handles incoming webhooks. Point your provider’s subscription URL at your Corsair HTTP handler (see [Overview](/plugins/twitterapiio/overview) for setup context and the exact URL shape).

<Info>
**New to Corsair?** See [webhooks](/concepts/webhooks) and [hooks](/concepts/hooks).
</Info>

## Webhook map

- `tweets`
  - `created` (`tweets.created`)
  - `filterMatch` (`tweets.filterMatch`)

## HTTP handler setup

```ts app/api/webhook/route.ts
import { processWebhook } from "corsair";
import { corsair } from "@/server/corsair";

export async function POST(request: Request) {
    const headers = Object.fromEntries(request.headers);
    const body = await request.json();
    const result = await processWebhook(corsair, headers, body);
    return result.response;
}
```

## Tweets

### Created

`tweets.created`

A monitored user posted a new tweet

**Payload**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `type` | `tweet.created` | Yes | — |
| `data` | `object` | Yes | — |
| `timestamp` | `string` | No | — |
| `userId` | `string` | No | — |
