Use **Spotify** through Corsair: one client, typed API calls, optional local DB sync.

**What you get:**

- 30 typed API operations
- 6 database entities synced for fast `.search()` / `.list()` queries

## Setup

<Steps>

<Step title="Install">
```bash
pnpm install @corsair-dev/spotify
```

</Step>

<Step title="Add the plugin">
<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { spotify } from '@corsair-dev/spotify';

export const corsair = createCorsair({
	// ... other config options,
	multiTenancy: false,
    plugins: [spotify()],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { spotify } from '@corsair-dev/spotify';

export const corsair = createCorsair({
	// ... other config options,
    multiTenancy: true,
    plugins: [spotify()],
});
```

See [Multi-tenancy](/concepts/multi-tenancy) for account isolation.

</Tab>
</Tabs>

</Step>

<Step title="Get credentials">
Follow [Get Credentials](/plugins/spotify/get-credentials) if you need help getting keys.

</Step>

<Step title="Store credentials">
<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --plugin=spotify
```

Use the key names documented in [Get Credentials](/plugins/spotify/get-credentials) (for example `api_key=`, `bot_token=`, or OAuth client fields).
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --plugin=spotify --tenant=<tenantId>
```

Store per-tenant secrets after you create the tenant record. See [Multi-tenancy](/concepts/multi-tenancy).
</Tab>
</Tabs>

</Step>

</Steps>

## Authentication

Each tab shows how to register the plugin for that authentication method. The default `authType` from the plugin does not need to appear in the factory call.

<Tabs>
<Tab title="OAuth 2.0 (Default)">

```ts corsair.ts
spotify()
```

Store credentials with `pnpm corsair setup --plugin=spotify` (see [Get Credentials](/plugins/spotify/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [OAuth 2.0](/concepts/oauth)

</Tab>
<Tab title="API Key">

```ts corsair.ts
spotify({
    authType: 'api_key',
})
```

Store credentials with `pnpm corsair setup --plugin=spotify` (see [Get Credentials](/plugins/spotify/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [API Key](/concepts/api-key)

</Tab>
</Tabs>


## Query synced data

Synced entities support `corsair.spotify.db.<entity>.search()` and `.list()`. See [Database](/plugins/spotify/database) for filters and operators.


## Example API calls

**Read-style (read):** `albums.get`

```ts
await corsair.spotify.api.albums.get({});
```


**Write-style (write):** `player.addToQueue`

```ts
await corsair.spotify.api.player.addToQueue({});
```


See the full list on the [API](/plugins/spotify/api) page. Use `pnpm corsair list --plugin=spotify` and `pnpm corsair schema <path>` locally to inspect schemas.
