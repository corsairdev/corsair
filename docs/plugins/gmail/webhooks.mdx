The Gmail plugin handles incoming webhooks. Point your provider’s subscription URL at your Corsair HTTP handler (see [Overview](/plugins/gmail/overview) for setup context and the exact URL shape).

<Info>
**New to Corsair?** See [webhooks](/concepts/webhooks) and [hooks](/concepts/hooks).
</Info>

## Webhook map

- `messageChanged` (`messageChanged`)

## HTTP handler setup

```ts app/api/webhook/route.ts
import { processWebhook } from "corsair";
import { corsair } from "@/server/corsair";

export async function POST(request: Request) {
    const headers = Object.fromEntries(request.headers);
    const body = await request.json();
    const result = await processWebhook(corsair, headers, body);
    return result.response;
}
```

## Events

## Message Changed

### Message Changed

`messageChanged`

A Gmail message was received, deleted, or had its labels changed

**Payload**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `message` | `object` | No | — |
| `subscription` | `string` | No | — |
| `event` | `any` | No | — |

**`webhookHooks` example**

```ts
gmail({
    webhookHooks: {
        messageChanged: {
            before(ctx, args) {
                return { ctx, args };
            },
            after(ctx, response) {
            },
        },
    },
})
```
