Use **Oura** through Corsair: one client, typed API calls, optional local DB sync, and incoming webhooks documented below.

**What you get:**

- 4 typed API operations
- 4 database entities synced for fast `.search()` / `.list()` queries
- 3 incoming webhook event types

## Setup

<Steps>

<Step title="Install">
```bash
pnpm install @corsair-dev/oura
```

</Step>

<Step title="Add the plugin">
<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { oura } from '@corsair-dev/oura';

export const corsair = createCorsair({
	// ... other config options,
	multiTenancy: false,
    plugins: [oura()],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { oura } from '@corsair-dev/oura';

export const corsair = createCorsair({
	// ... other config options,
    multiTenancy: true,
    plugins: [oura()],
});
```

See [Multi-tenancy](/concepts/multi-tenancy) for account isolation.

</Tab>
</Tabs>

</Step>

<Step title="Get credentials">
Follow [Get Credentials](/plugins/oura/get-credentials) if you need help getting keys.

</Step>

<Step title="Store credentials">
<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --plugin=oura
```

Use the key names documented in [Get Credentials](/plugins/oura/get-credentials) (for example `api_key=`, `bot_token=`, or OAuth client fields).
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --plugin=oura --tenant=<tenantId>
```

Store per-tenant secrets after you create the tenant record. See [Multi-tenancy](/concepts/multi-tenancy).
</Tab>
</Tabs>

</Step>

</Steps>

## Authentication

Each tab shows how to register the plugin for that authentication method. The default `authType` from the plugin does not need to appear in the factory call.

<Tabs>
<Tab title="API Key (Default)">

```ts corsair.ts
oura()
```

Store credentials with `pnpm corsair setup --plugin=oura` (see [Get Credentials](/plugins/oura/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [API Key](/concepts/api-key)

</Tab>
</Tabs>

## Webhooks

This plugin registers **3** webhook handler(s). Configure your provider to POST events to your Corsair HTTP endpoint, then use `webhookHooks` in the plugin factory for custom logic.

See [Webhooks](/plugins/oura/webhooks) for every event path and payload shape, and [Webhooks concept](/concepts/webhooks) for how to set up routing.


## Query synced data

Synced entities support `corsair.oura.db.<entity>.search()` and `.list()`. See [Database](/plugins/oura/database) for filters and operators.
