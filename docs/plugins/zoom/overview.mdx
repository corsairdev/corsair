Use **Zoom** through Corsair: one client, typed API calls, optional local DB sync, and incoming webhooks documented below.

**What you get:**

- 17 typed API operations
- 6 database entities synced for fast `.search()` / `.list()` queries
- 8 incoming webhook event types

## Setup

<Steps>

<Step title="Install">
```bash
pnpm install @corsair-dev/zoom
```

</Step>

<Step title="Add the plugin">
<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { zoom } from '@corsair-dev/zoom';

export const corsair = createCorsair({
	// ... other config options,
	multiTenancy: false,
    plugins: [zoom()],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { zoom } from '@corsair-dev/zoom';

export const corsair = createCorsair({
	// ... other config options,
    multiTenancy: true,
    plugins: [zoom()],
});
```

See [Multi-tenancy](/concepts/multi-tenancy) for account isolation.

</Tab>
</Tabs>

</Step>

<Step title="Get credentials">
Follow [Get Credentials](/plugins/zoom/get-credentials) if you need help getting keys.

</Step>

<Step title="Store credentials">
<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --plugin=zoom
```

Use the key names documented in [Get Credentials](/plugins/zoom/get-credentials) (for example `api_key=`, `bot_token=`, or OAuth client fields).
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --plugin=zoom --tenant=<tenantId>
```

Store per-tenant secrets after you create the tenant record. See [Multi-tenancy](/concepts/multi-tenancy).
</Tab>
</Tabs>

</Step>

</Steps>

## Authentication

Each tab shows how to register the plugin for that authentication method. The default `authType` from the plugin does not need to appear in the factory call.

<Tabs>
<Tab title="OAuth 2.0 (Default)">

```ts corsair.ts
zoom()
```

Store credentials with `pnpm corsair setup --plugin=zoom` (see [Get Credentials](/plugins/zoom/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [OAuth 2.0](/concepts/oauth)

</Tab>
</Tabs>

## Webhooks

This plugin registers **8** webhook handler(s). Configure your provider to POST events to your Corsair HTTP endpoint, then use `webhookHooks` in the plugin factory for custom logic.

See [Webhooks](/plugins/zoom/webhooks) for every event path and payload shape, and [Webhooks concept](/concepts/webhooks) for how to set up routing.


## Query synced data

Synced entities support `corsair.zoom.db.<entity>.search()` and `.list()`. See [Database](/plugins/zoom/database) for filters and operators.


## Example API calls

**Read-style (read):** `archiveFiles.list`

```ts
await corsair.zoom.api.archiveFiles.list({});
```


**Write-style (write):** `meetings.addRegistrant`

```ts
await corsair.zoom.api.meetings.addRegistrant({});
```


See the full list on the [API](/plugins/zoom/api) page. Use `pnpm corsair list --plugin=zoom` and `pnpm corsair schema <path>` locally to inspect schemas.
