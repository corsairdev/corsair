The Twilio plugin handles incoming webhooks. Point your provider’s subscription URL at your Corsair HTTP handler (see [Overview](/plugins/twilio/overview) for setup context and the exact URL shape).

<Info>
**New to Corsair?** See [webhooks](/concepts/webhooks) and [hooks](/concepts/hooks).
</Info>

## Webhook map

- `calls`
  - `status` (`calls.status`)
- `messages`
  - `received` (`messages.received`)
  - `status` (`messages.status`)

## HTTP handler setup

```ts app/api/webhook/route.ts
import { processWebhook } from "corsair";
import { corsair } from "@/server/corsair";

export async function POST(request: Request) {
    const headers = Object.fromEntries(request.headers);
    const body = await request.json();
    const result = await processWebhook(corsair, headers, body);
    return result.response;
}
```

## Events

## Calls

### Status

`calls.status`

Call status callback webhook from Twilio

**Payload**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `AccountSid` | `string` | Yes | — |
| `CallSid` | `string` | Yes | — |
| `CallStatus` | `queued \| initiated \| ringing \| in-progress \| completed \| busy \| failed \| no-answer \| canceled` | Yes | — |
| `ApiVersion` | `string` | No | — |
| `From` | `string` | No | — |
| `To` | `string` | No | — |
| `Direction` | `string` | No | — |
| `CallDuration` | `string` | No | — |
| `Duration` | `string` | No | — |
| `SipResponseCode` | `string` | No | — |
| `RecordingUrl` | `string` | No | — |
| `RecordingSid` | `string` | No | — |
| `RecordingDuration` | `string` | No | — |
| `CallbackSource` | `string` | No | — |
| `SequenceNumber` | `string` | No | — |
| `StirStatus` | `string` | No | — |

**`webhookHooks` example**

```ts
twilio({
    webhookHooks: {
        calls: {
            status: {
                before(ctx, args) {
                    return { ctx, args };
                },
                after(ctx, response) {
                },
            },
        },
    },
})
```
