Use **Slack** through Corsair: one client, typed API calls, optional local DB sync, and incoming webhooks documented below.

**What you get:**

- 41 typed API operations
- 5 database entities synced for fast `.search()` / `.list()` queries
- 9 incoming webhook event types

## Setup

<Steps>

<Step title="Install">
```bash
pnpm install @corsair-dev/slack
```

</Step>

<Step title="Add the plugin">
<Tabs>
<Tab title="Solo">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { slack } from '@corsair-dev/slack';

export const corsair = createCorsair({
	// ... other config options,
	multiTenancy: false,
    plugins: [slack()],
});
```
</Tab>
<Tab title="Multi-Tenant">
```ts corsair.ts
import { createCorsair } from 'corsair';
import { slack } from '@corsair-dev/slack';

export const corsair = createCorsair({
	// ... other config options,
    multiTenancy: true,
    plugins: [slack()],
});
```

See [Multi-tenancy](/concepts/multi-tenancy) for account isolation.

</Tab>
</Tabs>

</Step>

<Step title="Get credentials">
Follow [Get Credentials](/plugins/slack/get-credentials) if you need help getting keys.

</Step>

<Step title="Store credentials">
<Tabs>
<Tab title="Solo">
```bash
pnpm corsair setup --plugin=slack
```

Use the key names documented in [Get Credentials](/plugins/slack/get-credentials) (for example `api_key=`, `bot_token=`, or OAuth client fields).
</Tab>
<Tab title="Multi-Tenant">
```bash
pnpm corsair setup --plugin=slack --tenant=<tenantId>
```

Store per-tenant secrets after you create the tenant record. See [Multi-tenancy](/concepts/multi-tenancy).
</Tab>
</Tabs>

</Step>

</Steps>

## Authentication

Each tab shows how to register the plugin for that authentication method. The default `authType` from the plugin does not need to appear in the factory call.

<Tabs>
<Tab title="API Key (Default)">

```ts corsair.ts
slack()
```

Store credentials with `pnpm corsair setup --plugin=slack` (see [Get Credentials](/plugins/slack/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [API Key](/concepts/api-key)

</Tab>
<Tab title="OAuth 2.0">

```ts corsair.ts
slack({
    authType: 'oauth_2',
})
```

Store credentials with `pnpm corsair setup --plugin=slack` (see [Get Credentials](/plugins/slack/get-credentials) for field names). For OAuth, you typically store integration keys at the provider level and tokens per account or tenant.

More: [OAuth 2.0](/concepts/oauth)

</Tab>
</Tabs>

## Webhooks

This plugin registers **9** webhook handler(s). Configure your provider to POST events to your Corsair HTTP endpoint, then use `webhookHooks` in the plugin factory for custom logic.

See [Webhooks](/plugins/slack/webhooks) for every event path and payload shape, and [Webhooks concept](/concepts/webhooks) for how to set up routing.


## Query synced data

Synced entities support `corsair.slack.db.<entity>.search()` and `.list()`. See [Database](/plugins/slack/database) for filters and operators.


## Example API calls

**Read-style (read):** `channels.get`

```ts
await corsair.slack.api.channels.get({});
```


**Write-style (destructive):** `channels.archive`

```ts
await corsair.slack.api.channels.archive({});
```


See the full list on the [API](/plugins/slack/api) page. Use `pnpm corsair list --plugin=slack` and `pnpm corsair schema <path>` locally to inspect schemas.
