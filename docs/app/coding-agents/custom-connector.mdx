---
title: Custom connector
description: Connect any HTTP MCP client to Corsair with a URL and bearer token.
---

You can connect a **custom MCP client** to Corsair through the [dashboard](#in-the-dashboard) or [in code](#in-code).

## In the dashboard {#in-the-dashboard}

1. Sign in at [app.corsair.dev](https://app.corsair.dev).
2. Open **Dashboard** and click **New instance**.
3. Open your instance and click **Add Plugin** to enable an integration (e.g. GitHub, Slack).
4. Click **New User** to add a tenant — enter a tenant ID and click **Save**.
5. Connect the integration for that tenant: click the cell where the tenant row meets the plugin column, then sign in with OAuth or enter credentials.
6. Click the **tenant name** in the first column to open the side panel.
7. Copy the **MCP URL** and create an **API key** (**New key** → name → **Create** → copy the secret once).
8. Paste the URL and secret into your connector (`url` or `serverUrl`; auth is `Authorization: Bearer <secret>`). OAuth discovery URLs are also shown in the tenant panel if your client supports them.

## In code {#in-code}

### Generic config

```json
{
  "type": "http",
  "url": "https://api.corsair.dev/mcp/<instanceId>?tenantId=<tenantId>",
  "headers": {
    "Authorization": "Bearer <tenant-api-key-secret>"
  }
}
```

### With the SDK

```typescript
const key = await inst.tenant("your-tenant-id").mcpKeys.create("my-connector");
```

## Try it

Run a prompt in your agent.

If you connected GitHub, try:

```
Use Corsair to star the Corsair repo on GitHub
```

<Warning>
Do not use your developer API key from **API Keys** in the sidebar as the MCP bearer token unless you intend to use the developer-key flow with `tenantId` in the URL.
</Warning>
